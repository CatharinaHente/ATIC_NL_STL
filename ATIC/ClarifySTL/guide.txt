ClarifySTL Framework: Usage Guide
Step A. Training the Incompleteness and Ambiguity Detection Models

1) Download the Base Model

Download the LLaMA 3-8B model and prepare the model directory (e.g., /models/LLaMA-3-8B).

2) Prepare the Dataset

Use the AmbiEval_A and AmbiEval_B datasets.

Convert the datasets to Alpaca format (JSONL) with fields instruction, input, output.

3) Place the processed datasets in:

/ClarifySTL/LLaMA-Factory-main/data/


4) Train the Models

In /ClarifySTL/LLaMA-Factory-main/examples/train_lora/ there are two training scripts:

Incompleteness_detector.sh – trains the incompleteness detection model

Ambiguity_detector.sh – trains the ambiguity detection model

Example commands:

bash Incompleteness_detector.sh \
  --model_name_or_path /models/LLaMA-3-8B \
  --dataset /ClarifySTL/LLaMA-Factory-main/data/AmbiEval_A.jsonl \
  --output_dir /ClarifySTL/output/IncompletenessDetector

bash Ambiguity_detector.sh \
  --model_name_or_path /models/LLaMA-3-8B \
  --dataset /ClarifySTL/LLaMA-Factory-main/data/AmbiEval_B.jsonl \
  --output_dir /ClarifySTL/output/AmbiguityDetector


Parameters:

model_name_or_path: path to the base LLaMA model

dataset: path to the training dataset

output_dir: path to save the trained model

Step B. Running the ClarifySTL Framework in /ClarifySTL/LLM_agent_framework/clarify.py

Prepare Requirement File

Write the natural language requirement(s) in a .txt file, one requirement per file.

Example:

/ClarifySTL/requirements/sample_requirement.txt


Run the Framework

Execute the orchestrator:

python clarify_stl_orchestrator.py \
  --input /ClarifySTL/requirements/sample_requirement.txt \
  --output /ClarifySTL/results/clarified_result.json \
  --human   # optional, enables human-in-the-loop interaction


Parameters:

--input: path to the requirement text file

--output: path to the output JSON file

--human: enable human interaction; if omitted, the framework runs in automated mode (requires auto_responder_llm)

Output Description

The output JSON file contains:

{
  "original": "...",                  // original requirement
  "after_incompleteness": "...",      // requirement after incompleteness clarification
  "incompleteness_history": [...],    // history of incompleteness clarification steps
  "after_ambiguity": "...",           // requirement after ambiguity clarification
  "ambiguity_history": [...],         // history of ambiguity clarification steps
  "final_stl": "..."                  // final STL formula generated by LLM
}
